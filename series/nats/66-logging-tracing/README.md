# 66 — Logging & Tracing

> **Type:** How-To  
> **Phase:** Production & Operations

## Overview

Visibility into what is happening inside your NATS cluster and between your services is vital for debugging. This lesson covers server logging and distributed tracing for message flows.

## 1. Server Logging

Configure the level of detail in the server output.

`server.conf`:
```
debug: true        # Log connection/subscription info
trace: true        # Log raw protocol traffic (Warning: Very Verbose!)
logtime: true      # Include timestamps
logfile: "/var/log/nats-server.log"
```

### Signal Handling
You can toggle logging levels at runtime:
- `kill -USR2 <pid>` : Toggles `debug` logging on/off.
- `kill -USR1 <pid>` : Re-opens log files (for rotation).

## 2. Tracing Messages (Correlation ID)

Since NATS messages are independent, you must inject a **Correlation ID** into headers to trace a request through multiple services.

```go
// Service 1: Inject ID
msg := &nats.Msg{
    Subject: "orders.created",
    Header:  nats.Header{},
}
traceID := uuid.New().String()
msg.Header.Set("X-Trace-ID", traceID)
nc.PublishMsg(msg)

// Service 2: Extract and forward
nc.Subscribe("orders.created", func(msg *nats.Msg) {
    traceID := msg.Header.Get("X-Trace-ID")
    log.Printf("[%s] Processing order events", traceID)
    
    // Pass same ID to downstream requests
    nextMsg := &nats.Msg{Subject: "payments.charge", Header: nats.Header{}}
    nextMsg.Header.Set("X-Trace-ID", traceID)
    nc.PublishMsg(nextMsg)
})
```

## 3. OpenTelemetry Integration

NATS doesn't have native OTel support in the server (yet), but you can use the OpenTelemetry Go SDK to wrap your producers/consumers.

```go
import "go.opentelemetry.io/otel"

func publish(ctx context.Context, nc *nats.Conn, subject string, data []byte) {
    tr := otel.Tracer("nats-client")
    ctx, span := tr.Start(ctx, "nats publish "+subject)
    defer span.End()

    msg := nats.NewMsg(subject)
    msg.Data = data
    // Propagate context into headers
    otel.GetTextMapPropagator().Inject(ctx, nats.HeaderCarrier(msg.Header))
    
    nc.PublishMsg(msg)
}
```

## Tools for Trace Visualization
- **Jaeger/Tempo:** For viewing end-to-end spans.
- **Service Map:** Generated by Grafana or Jaeger to see which services talk to which subjects.
