> **Type:** Tutorial

## What is Claude Code?

[Claude Code](https://docs.anthropic.com/en/docs/about-claude/models/claude-code) is Anthropic's agentic coding CLI. It can read your codebase, write test files, run commands, and iterate based on results — all from a single prompt.

## Setup

```bash
# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Set your API key
export ANTHROPIC_API_KEY="sk-ant-..."
```

## Workflow: Generate tests from a feature description

```bash
# Start Claude Code in the playwright-series directory
claude-code

# Example prompts:
> "Look at test-app/pages/forms.html and generate a Playwright TypeScript 
   test file for the signup form. Cover: happy path, each validation error, 
   and password strength meter."

> "Run the tests you just created and fix any failures."
```

Claude Code will:
1. Read `forms.html` to understand the component
2. Write `tests/forms.spec.ts` with full test coverage
3. Run `npx playwright test tests/forms.spec.ts`
4. Read the failure output and fix the tests
5. Re-run until all pass

## Example generated test (excerpt)

```typescript
// Generated by Claude Code
import { test, expect } from '@playwright/test';

test.describe('Signup Form', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('/pages/forms.html');
  });

  test('submits successfully with valid data', async ({ page }) => {
    await page.getByTestId('signup-name').fill('Alice Johnson');
    await page.getByTestId('signup-email').fill('alice@example.com');
    await page.getByTestId('signup-password').fill('SecurePass1!');
    await page.getByTestId('agree-terms').check();
    await page.getByTestId('signup-submit').click();
    await expect(page.getByTestId('form-result')).toContainText('Account created');
  });

  test('shows error when name is too short', async ({ page }) => {
    await page.getByTestId('signup-name').fill('A');
    await page.getByTestId('signup-submit').click();
    await expect(page.getByTestId('name-error')).toBeVisible();
  });
});
```

## ✅ Positive test cases

| # | Scenario | Expected result |
|---|----------|-----------------|
| 1 | Prompt generates test file | Valid TypeScript file created |
| 2 | Generated tests run | All pass against test app |
| 3 | Claude fixes a failing test | Iterates until green |

## ❌ Negative test cases

| # | Scenario | Expected result |
|---|----------|-----------------|
| 1 | Ambiguous prompt | Claude asks clarifying question |
| 2 | Non-existent data-testid referenced | Test fails; Claude reads DOM and corrects selector |
| 3 | No ANTHROPIC_API_KEY set | CLI exits with clear auth error |

## Tips for effective prompts

- **Give context**: "Look at `test-app/pages/auth.html` before generating"
- **Specify scope**: "Cover positive + negative cases for each field"
- **Ask for iteration**: "Run the tests after writing them and fix any failures"
- **Use data-testid**: The test app's `data-testid` attributes help Claude pick stable selectors
